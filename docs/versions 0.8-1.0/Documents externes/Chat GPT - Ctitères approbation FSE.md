Il est important de distinguer deux notions :

    Le FSE+ (Fonds social européen Plus) ne délivre pas de certification officielle pour les applications d'émargement.

    En revanche, une application doit permettre au porteur de projet de satisfaire aux exigences de contrôle et d'audit du FSE+. C'est lors des contrôles que la solution est évaluée, et non par une procédure d'homologation préalable.

Les principales exigences qu'une application d'émargement numérique doit respecter sont les suivantes :
1. Identification certaine des participants

L'application doit permettre d'identifier clairement :

    le participant (nom, prénom),

    l'intervenant,

    la structure organisatrice.

2. Traçabilité complète

Chaque émargement doit enregistrer :

    la date,

    l'heure,

    la durée,

    l'action réalisée (formation, accompagnement, atelier, etc.),

    toute modification éventuelle (journal d'audit).

Les contrôleurs doivent pouvoir vérifier qu'aucune signature n'a été modifiée après coup.
3. Signature électronique fiable

La signature doit être :

    personnelle,

    liée au signataire,

    horodatée,

    conservée avec les preuves techniques.

En pratique, une signature électronique simple peut être admise si elle permet d'apporter des preuves suffisantes, mais une signature répondant au règlement eIDAS offre un niveau de sécurité supérieur.
4. Conservation des preuves

L'application doit conserver :

    les feuilles d'émargement,

    les métadonnées,

    les horodatages,

    les journaux de modification,
    pendant toute la durée réglementaire de conservation des pièces justificatives du projet.

5. Respect du RGPD

L'application doit notamment prévoir :

    la protection des données personnelles,

    l'hébergement sécurisé,

    la gestion des droits d'accès,

    la possibilité d'exporter ou supprimer les données selon les obligations légales.

6. Export des justificatifs

En cas de contrôle, il doit être possible d'exporter facilement :

    les feuilles d'émargement en PDF,

    les listes de présence,

    les historiques,

    les journaux d'audit.

7. Contenu des feuilles d'émargement

Les justificatifs doivent généralement faire apparaître :

    le nom de l'action,

    la date,

    les horaires de début et de fin,

    les noms des participants,

    le nom de l'intervenant,

    les signatures,

    les mentions et logos du financement européen lorsque l'action est cofinancée par le FSE+, conformément aux obligations de publicité. Les exemples de documents publiés par des gestionnaires FSE reprennent ces éléments.

8. Intégrité des données

Une bonne solution doit empêcher :

    la suppression d'un émargement sans trace,

    la modification des signatures,

    la falsification des dates ou horaires.

Bonnes pratiques attendues lors d'un contrôle FSE+

Une application sera généralement considérée comme satisfaisante si elle offre notamment :

    ✓ authentification des utilisateurs ;

    ✓ horodatage automatique ;

    ✓ signature électronique individuelle ;

    ✓ journal d'audit inaltérable ;

    ✓ archivage sécurisé ;

    ✓ export PDF des feuilles d'émargement ;

    ✓ conformité RGPD ;

    ✓ conservation des preuves techniques.

En résumé, il n'existe pas de liste officielle d'applications "approuvées FSE+". La conformité est appréciée au regard de la capacité de la solution à produire des preuves fiables lors d'un contrôle administratif ou d'un audit.
